# ⚽ فكرة المسابقة

هي مسابقة تفاعلية في معلومات كرة القدم، تُقام بين لاعبين (2 ضد 2) يمثلون الأندية المشاركة في محافظة إدلب، وتهدف لاختبار المعرفة الكروية بطريقة حماسية تشبه برامج المسابقات التلفزيونية.

تعتمد المسابقة على نظام **خروج المغلوب**، حيث تتنافس الفرق في مواجهات مباشرة، والفائز يتأهل للمرحلة التالية حتى الوصول إلى النهائي وتحديد البطل.

## 🎮 آلية اللعب

كل مباراة تتكون من عدة فقرات متنوعة، وكل فقرة تقيس نوع مختلف من المعرفة والسرعة:

- **ماذا تعرف:** أسئلة عامة في كرة القدم
- **المزاد:** منافسة على الإجابة خلال وقت محدد (مثلاً 30 ثانية)
- **الجرس:** سرعة البديهة والضغط على الجرس للإجابة
- **المستحيل:** أسئلة صعبة جداً تحتاج معرفة عميقة
- **من أنا:** تخمين لاعب أو شخصية كروية من خلال تلميحات

## 🧠 نظام النقاط

- يحصل اللاعبون على نقاط عند الإجابات الصحيحة
- يمكن زيادة أو إنقاص النقاط حسب الأداء
- يتم تحديث النتائج مباشرة أثناء المباراة لزيادة الحماس والتفاعل

## 📺 تجربة العرض

تُعرض المباراة على شاشة مباشرة أمام الجمهور، حيث يتم إظهار:

- أسماء اللاعبين وصورهم
- النقاط الحالية
- الفقرة الحالية
- الوقت (في بعض الفقرات)

كل ذلك يتم بشكل لحظي، مما يعطي إحساس برنامج مسابقات حقيقي.

## 🏆 الهدف النهائي

الوصول إلى المباراة النهائية، حيث يتنافس أفضل فريقين، ويتم تتويج الفائز بـ:

- كأس البطولة
- جوائز مادية ومعنوية
- لقب بطل المسابقة

أمرك يا سيدي 😈… خلّينا نمشيها **clean + مرتب + production-ready** بدون لف ودوران.

## 🏟 Match Schema (TypeScript)

```ts
export type Match = {
  _id?: string;

  playerA: {
    name: string;
    image: string;
    rank: number;
  };

  playerB: {
    name: string;
    image: string;
    rank: number;
  };
  round: "ROUND_16" | "ROUND_8" | "QUARTER_FINAL" | "SEMI_FINAL" | "FINAL";

  segment:
    | "WHAT_DO_YOU_KNOW"
    | "AUCTION"
    | "BUZZER"
    | "IMPOSSIBLE"
    | "WHO_AM_I";
  timer?: number | null;
  isFinal: boolean;
  status: "SETUP" | "FINISHED";

  createdAt?: Date;
  updatedAt?: Date;
};
```

```yaml
            segment: "AUCTION"

    }
```

# 📏 8. Rules & Best Practices

## 🧠 Architecture Rules

- Single Source of Truth = matchState
- لا تفصل backend
- لا تعمل duplication للstate

## 🔌 WebSocket Rules

- كل update لازم يمر عبر السيرفر
- لا تحدث UI مباشرة بدون socket
- debounce updates إذا لزم

## 🧾 Form Rules

- كل input مربوط بـ React Hook Form
- validation عبر Zod فقط
- no manual validation

## 🗄️ Data Rules

- rank يبدأ 0 دائمًا
- timer فقط للمزاد
- winner فقط في النهاية

## 🎛️ UX Rules

- الفورم يقسم لعدة اجزاء
  - create: { playerA: { name, image, rank },playerB: { name, image, rank }, round, isFinal, segment }
  - staps: { id, data: { playerA: { rank }, playerB: { rank }, segment } }
    - اذا كان هناك في segment المزاد اضهر العداد
    - عند فتح المبارة يجب فتح مربعين فيهما
      - names, rank, image
      - بالاضافة ل الى ان الرانك تحته ازرار موجبة وسالبة من 1 -5 وهناك زر الى input للتحكم اكبر في المدخلات
      - مع زرالتقدم والرجوع
      - طبعا زر التقدم هو تحديث segment
      - في المرحلة الاخيرة الزر هو انهاء المبارة
    - بعد انتهاء المبارة من نفس التصميم سوف نصمم صفحة للعرض الفائز في الجولة بحسب نقاطه
- كل step بسيط وواضح
- no overload في الفورم
- live preview دائمًا
- 🧠 Simplicity First
    - أي عنصر غير ضروري = حذف
    - no extra labels
- ⚡ Speed
    - أقل عدد clicks ممكن
    - لا يوجد انتظار
    - no duplication
- 🎯 Clarity
    - اللاعب A يسار دائماً
    - اللاعب B يمين دائماً
- 🧩 Predictability
    - نفس layout بكل الفقرات
    - لا تغيير مفاجئ
🎮 Focus Mode
- الشاشة بدون تشتيت فقط:لاعبين نقاط
